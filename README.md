# 🚀 Simplicity is the Ultimate Sophistication

안녕하세요, **단순함의 본질에 집중하는 개발자 이진우**입니다.  
복잡한 문제를 만났을 때 기술을 더하기보다, **비즈니스 가치를 극대화할 수 있는 가장 단순하고 명확한 해답**을 설계하는 과정을 즐깁니다.

---

### 💡 Engineering Philosophy

* **Simplicity over Complexity**: 오버 엔지니어링을 경계합니다. 동료 누구나 1분 안에 이해할 수 있는 명확한 로직이 가장 효율적인 코드라고 믿습니다.
* **Evidence-based Design**: "왜 이 기술인가?"에 대해 막연한 선호도가 아닌 **데이터와 제약 조건(Storage, 성능, 비용)**을 근거로 의사결정합니다.
* **Collaborative Growth**: 기록을 무엇보다 중요시하여, 의사결정 과정을 Issue와 PR에 상세히 기록하고 팀의 커뮤니케이션 비용을 낮추는 데 집중합니다.

### 📂 Featured Projects

#### 1. [CREATIVE AI](https://github.com/enderpawar/CREATIVE_AI) | 노드 기반 비주얼 AI 파이프라인 에디터
> **"복잡한 그래프 구조를 안정적이고 가볍게 관리하는 최적화 솔루션 구현"**

* **Challenge**: 노드 간 순환 참조 발생 시 시스템 무한 루프 및 브라우저 성능 저하 위험.
* **Solution**: **DFS(Depth-First Search) 알고리즘**을 적용한 순환 참조 사전 감지 로직으로 그래프 무결성 확보.
* **Key Results**: 
    * 데이터 직렬화 최적화를 통해 브라우저 Local Storage 점유율 **약 60% 절감**.
    * 무결성 검증 로직 최적화로 대규모 노드 환경에서도 **지연 시간 최소화**.
* **Insight**: 브라우저 환경의 물리적 제한을 인지하고, 효율적인 데이터 구조 설계가 UX에 미치는 영향을 깊이 이해하게 되었습니다.

#### 2. [TRADE BUILDER](https://github.com/Trade-Builder/Trade-Builder-Client) | 전략적 자산 운용 자동화 시스템
> **"방대한 실시간 데이터를 핵심 지표로 필터링하여 시스템 부하 감소"**

* **Challenge**: 초당 수백 건의 시장 데이터 유입 시 클라이언트 렌더링 병목 현상 발생.
* **Solution**: 데이터 흐름을 단순화하고, 필요한 **핵심 지표(Signal)만 추출하는 필터링 알고리즘** 설계.
* **Key Results**: 
    * 데이터 파이프라인 효율화로 불필요한 리렌더링 **40% 감소**.
    * 메모리 사용량 **약 25% 절감**을 통해 저사양 기기에서도 안정적인 트레이딩 환경 제공.
* **Insight**: 복잡한 비즈니스 로직을 최소 단위(Atomic)로 분리함으로써 코드의 재사용성과 유지보수성을 동시에 확보했습니다.

### 📈 Daily & Collaborative Habit

* **Continuous Learning**: 매일 알고리즘 문제 풀이와 기록을 통해 논리적 사고의 끈을 놓지 않습니다.
